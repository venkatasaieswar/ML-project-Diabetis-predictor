
# TeamXX — Clinical Prediction Dashboard (Assignment 2)

This repository contains the code and dashboard for the Diabetes prediction assignment.

Important: CSV data files and generated outputs are large and are excluded from the repository. To run the pipeline and the dashboard locally, download the dataset CSVs and place them under the `csv/` folder (same filenames as used in the original project). The pipeline scripts will write model artifacts and figures into the `outputs/` folder.

Quick start

1. Create and activate a Python virtual environment (recommended):

   python3 -m venv .venv
   source .venv/bin/activate

2. Install dependencies (examples):

   pip install -r requirements.txt

3. Place the CSV files in the `csv/` directory. The repo expects files such as `patients.csv`, `encounters.csv`, `observations.csv`, `medications.csv`, `conditions.csv`, etc.

4. Run the preprocessing and training pipeline (examples):

   python3 src/04_aggregate.py
   python3 src/05_split_and_label.py
   python3 src/07_train.py
   python3 src/08_continual.py
   python3 src/09_explain_eda.py

5. Start the dashboard (reads from `outputs/`):

   streamlit run TeamXX_Assignment2_dashboard.py

Notes
- Keep the `csv/` folder locally — do not commit it to GitHub (it is listed in `.gitignore`).
- The `outputs/` folder is generated by the scripts and also ignored by Git. If you want to share artifacts, create a separate artifact archive (zip) or push to a release storage location.

Storing outputs in the repository

This project stores light-weight outputs (trained models, figures, and summary CSVs) under `outputs/`. If your outputs directory is small (under a few hundred MB), you may include it in the repo for convenience. To add outputs to the repository, ensure `outputs/` is not commented out in `.gitignore` (we currently leave it enabled by default), then commit and push. Large raw CSVs in `csv/` should remain local only.

Running everything with uv (uv is a small helper wrapper used in this project)

1. To install uv, if not already present, you can use pip:

   pip install uv

2. Run the pipeline using uv (example sequence):

   uv run python3 src/04_aggregate.py
   uv run python3 src/05_split_and_label.py
   uv run python3 src/07_train.py
   uv run python3 src/08_continual.py
   uv run python3 src/09_explain_eda.py

3. Start dashboard:

   uv run streamlit run TeamXX_Assignment2_dashboard.py

If you prefer not to use uv, run the same commands directly with python/streamlit.
